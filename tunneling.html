Tunneling Overview

Since VPNs allocate different data responsibilities to the different OSI layers, there are many ways to combine the data in such a way that secures it from tampering.
The process of combining all the data from the layers into a sendable packet is called Tunneling; this is executed through a procedure called a Tunneling Protocol.

There are many Tunneling Protocols that have been created since the dawn of the Internet which is why there are so many VPN services today. These Tunneling Protocols not only combine information from the OSI layers, but also additional information to ensure security and integrity of the data being sent.

For this reason, Tunneling Protocols can occur at any layer of the OSI.

Point-to-Point Tunneling Protocol (PPTP)

As one of the oldest Tunneling Protocols, PPTP was a Data Link Layer Tunneling Protocol used for dial up networks back in the early 2000s. It utilized Microsoft Challenge Handshake Protocol(MS-CHAP) to set up a connection and encrypt messages. It was very easy to connect with PPTP as MS-CHAP utilized a 40 bit Encryption Key based on the user's password. While not used very much anymore, it inspired other, better Protocols to be created(Bates 65) (Anon).

Layer 2 Tunneling Protocol (L2TP)

L2TP was a successor of PPTP as another Data Link Layer Tunneling Protocol. L2TP focused on speed and data integrity above anything else(Conrad, et al.). The encapsulation at the Data Link Layer performed two operations: UDP packaging and IP packaging. User Datagram Protocol(UDP) is protocol that packages the data with a binary value called a "Check Sum" to easily verify tampered data(Fox and Khan Academy).

For example:
The CheckSum is 2 bytes. The Data is 4 bytes and is broken up into 2 byte blocks. The blocks are XORed together producing the CheckSum. The CheckSum is appended to the packet and is sent to the receiver. The receiver can re-calculate the CheckSum from the data in the packet to verify if they match.

Internet Protocol Security (IPSec)

IPSec occurs at the Network Layer, also known as the IP Layer. IPSec can be flexible in providing high quality data integrity and encryption because it apply an Authentication Header (AH) and Encapsulating Security Payload (ESP) Protocol (IBM).

AH provides authentication by utilizing Hashing Algorithms to create digital signatures (IBM).
Some of these Algorithms include:

HMAC_MD5
HMAC_SHA1
HMAC_SHA2_256_128
HMAC_SHA2_384_192
HMAC_SHA2_512_256
AES_GMAC_128
AES_GMAC_256

ESP provides encryption with the Algorithms (IBM):
AES_CBC KeyLength 128 AES_CBC KeyLength 256

HMAC:Hashed Message Authentication Code
MD:Merkle-Damgard
SHA:Secure Hash Algorithm
AES:Advanced Encryption Standard
GCM:Galois Message Authentication Code
CBC:Cipher Block Chaining



IPSec uses many other protocols that impact the IP address header added to the packet. IPSec has different modes that change how much information it adds to the packet.

Tunnel Mode (more secure):

Entire packet is encrypted and authenticated. After encryption, a new IP header is added over the packet.

PSec Tunnel Mode Packet, Bates 81



Transport Mode (faster):

Only Payload (Data) is encrypted and authenticated

IPSec + L2TP

Since L2TP is primarily an authentication Tunneling Protocol, it is ofent paired with IPSec for its security. Not only that, but L2TP wraps an IP package around the packetm which allows seamless application of IPSec Protocols (Harrington).
